SQL Challenge


SELECT
    mov_title AS "Movie Title"
    mov_year AS Year  "Year Released"
    mov_time AS "Running Time"
    mov_lang AS "Language"
    rating.rev_stars AS "Rating"
FROM movie
JOIN rating ON rating.mov_id = movie.mov_id
WHERE rating.rev_stars > 8
AND
    mov_year BETWEEN 1950 AND 2000
AND
    mov_time > 120
ORDER BY mov_year ASC

This challenge was to test my SQL skills by querying a movies database using a sub-query

My query retrieves all films with a rating of more than 8, that where made between 1950 and 2000 and running time of over 2 hours.

To do this I joined the the movie table with the rating table using their matching primary key.

JavaScript Reflection


form.addEventListener('submit', e =>  {
    e.preventDefault(); //Prevents browser reloading on user submit
    //validation
    let userEmail = form.email.value;
    const regex = /^[a-zA-Z0-9\.!#$%&'*+/=?^_`{|}~-]{2,64}@[a-zA-Z0-9-]+\.[a-zA-Z]{2,8}(\.[a-zA-Z]{2,8})?$/;
    //On submission of email link image to email address
    if(regex.test(userEmail)){
        emailModal.style.display = 'none';
        linkToEmailButton.innerHTML = 'email added...';
        linkToEmailButton.disabled = true;
        setTimeout(() => {
            linkToEmailButton.disabled = true;
            linkToEmailButton.innerHTML = 'link to email';
            getPhoto();
        }, 2000);
        let newEmailAddress = userEmail;
        
        if(! newPicBox[newEmailAddress]) { //If no array with that name in newPicBox make one and push pic to it 
            newPicBox[newEmailAddress] = [];
            newPicBox[newEmailAddress].push(result);
        } else {
            newPicBox[newEmailAddress].push(result);
        }
    } else {
        alert("That's not a valid email!");
    }
});

The challenge that I was set was to retrieve a random image from an API (I used upsplash) and display it to the user with a button to choose another image.

The user could the save this image by linking it to an email address and this is what this code block demonstrates.

If the user email is valid and therefore passes the regex test the modal that they have inputted their email into disappears and the image on the page is refreshed.

The email address is temporarily stored in the newEmailAddress variable and if an array with this name doesn't already exist in the newPicBox object then one is created and the picture URL is pushed into it. Otherwise if an array with this name does exist then the picture is pushed to it.

In addition the user is able to link to numerous email address and then when the click each address a modal window displays all of the images that are linked to that email address.

I found this challenge very useful as it really helped me to understand objects and arrays, how to sort and retrieve data from them.

Homepage Reflection


@mixin tooltip-main {
    background-color: $dark-blue;
    color: $white;
    position: absolute;
    display: none;
    max-width: 260px;
    text-align: center;
}
.show {
    .tooltip {
        @include tooltip-main;
        padding: 20px;
        p {
            margin-top: 1rem;
            font-size: 0.875rem;
            font-weight: $light;
        }
        &::after { //Creates the dropdown arrow
            content: '';
            border-left: solid transparent 30px;
            border-right: solid transparent 30px;
            border-top: solid $dark-blue 30px;
            bottom: -29px; //Makes it sit flush with the bottom of the element
            height: 0;
            left: 37.5%; //Centers the arrow
            position: absolute; //Makes it appear below
            margin-left: 0px;
        }
    } 
    &:hover .tooltip {
        display: block;
        pointer-events: auto;
    }
}

This is an example of using SASS and CSS to create a tooltip that appears when you hover over a clients logo.

The top section is an example of a mixin, which is a re-useable block of code which helps to reduce time.

There are also examples of variables, in this case for colors used within the site. This is helpful because if at any point there is to be a change to a sites color scheme, the variables can be changed rather than every instance of the color being implemented.

The second block contains an example of using psuedo elements, in this case using transparent borders to create an arrow shape that protrudes out of the elements container.

It also demonstrates the use of nesting code within SASS and commenting code which is useful when you come back to a project and have no idea what anything does anymore!
